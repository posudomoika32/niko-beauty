# NikoBeauty

#  **NikoBeauty — онлайн‑сервис и запись в бьюти-салон**

##  Краткое описание  
**NikoBeauty** — это современный веб-сервис для бьюти-салона, который позволяет пользователям быстро и удобно знакомиться с перечнем услуг и записываться к мастерам онлайн. Платформа предоставляет детализированный каталог процедур (парикмахерские услуги, ногтевой сервис, косметология, SPA) с описанием, ценами и длительностью. Сервис ориентирован на комфорт клиента, интуитивно понятный интерфейс и экономию времени при планировании визита. NikoBeauty помогает подчеркнуть естественную красоту, делая процесс записи максимально простым.  

##  Целевая аудитория  
- Клиенты, регулярно посещающие салоны красоты  
- Люди, которым нужна удобная онлайн-запись без звонков администратору  
- Новые посетители, выбирающие салон по отзывам и портфолио работ  
- Мастера салона, отслеживающие свое расписание  
- Администраторы, управляющие записями, услугами и базой клиентов  

##  Ключевой функционал  
- Регистрация и аутентификация пользователей  
- Личный кабинет с историей визитов и редактированием контактных данных  
- Каталог услуг с удобным разделением по категориям (Волосы, Ногти, Лицо, Тело)  
- Детальная страница услуги (описание, цена, время выполнения, фото)  
- Онлайн-запись (выбор услуги, мастера, даты и времени)  
- Избранные мастера или процедуры  
- Админ‑панель для управления услугами, расписанием и клиентами  
- Проверка уникальности логина и телефона при регистрации  
- Динамическая загрузка свободного времени и данных с сервера

#  Технологический стек

## Frontend
### **1. Чистый JavaScript**
**Почему чистый JavaScript**
- **Нет лишней сложности.** Для сайта салона красоты с функционалом записи классического JS более чем достаточно, это избавляет от оверхеда крупных SPA-фреймворков (React/Vue).  
- **Меньше зависимостей.** Приложение загружается мгновенно, что критически важно для мобильных пользователей, желающих быстро записаться.  
- **Прямой контроль над DOM.** Удобно реализовывать интерактивные элементы: выбор времени в календаре, фильтрацию услуг, личный кабинет.

### **2. HTML + CSS**
**Почему HTML и CSS**
- **Простота и скорость разработки.** Легковесный подход без тяжелых библиотек.  
- **Полный контроль над версткой.** Позволяет создать эстетичный, пиксель-перфект дизайн, что крайне важно для бьюти-сферы.  
- **Легко поддерживать.** Любой разработчик сможет быстро внести изменения в интерфейс.

### **3. Fetch API**
**Почему Fetch API**
- **Современный стандарт.** Идеально подходит для асинхронной загрузки свободных слотов времени или отправки формы записи без обновления страницы.  
- **Не требует библиотек.** Избавляет от необходимости тащить в проект Axios.

##  Backend
### **1. Node.js (Express)**
**Почему Node.js**
- **Высокая скорость обработки запросов.** Идеально подходит для REST API и обработки одновременных запросов на бронирование времени.  
- **Один язык на фронте и бэке.** JavaScript везде — единая экосистема и простота поддержки.  
- **Минималистичный и гибкий.** Позволяет быстро создать надежный бэкенд для маршрутов регистрации, каталога услуг и системы бронирования.  

### **2. MySQL**
**Почему MySQL**
- **Реляционная структура.** Система салона — это строгие связи: Пользователь -> Запись -> Услуга -> Мастер. MySQL идеально решает эту задачу.  
- **Надёжность и консистентность.** Исключает ситуации двойной записи на одно и то же время благодаря транзакциям.  
- **Поддержка хостингов.** Легко развернуть на любой современной платформе (Railway, PlanetScale, VPS).

### **3. bcrypt**
**Почему bcrypt**
- **Безопасное хранение паролей.** Хеширование с использованием соли защищает данные клиентов.  
- **Защита от brute‑force атак.** Медленный алгоритм предотвращает перебор паролей.  
- **Простая интеграция.** Отличная совместимость с экосистемой Node.js.

### **4. express-session**
**Почему express-session**
- **Удобная авторизация.** Сохранение состояния сессии для безопасного перехода клиента к процессу бронирования.  
- **Безопасность.** В сессии хранится только ID и роль пользователя, без чувствительных данных.

##  Инфраструктура
- **Frontend** — Vercel / GitHub Pages  
- **Backend** — Render / Railway / VPS  
- **База данных** — MySQL (PlanetScale / Railway / локальный сервер)

#  Визуальное проектирование интерфейса

## **Страница: Главная (/)**
**Тип доступа:** публичный  

### Основные блоки:
1. **Шапка:** логотип NikoBeauty, меню (Главная, Услуги, Мастера, Войти), имя пользователя  
2. **Блок приветствия:** эстетичный баннер, кнопка «Записаться онлайн»  
3. **Секция “Популярные услуги”:** 6 карточек (например, сложное окрашивание, смарт-педикюр)  
4. **Футер:** контакты, адрес салона, соцсети, режим работы  

**Динамический контент:** `GET /api/services?popular=true`

---

## **Страница: Аутентификация (/auth)**
**Тип доступа:** публичный  

### Основные блоки:
- Форма входа  
- Форма регистрации  
- Поля: имя, логин, пароль, телефон  
- Проверка уникальности номера телефона (для связи администратора)  
- Сообщения об ошибках  

---

## **Страница: Каталог услуг (/services)**
**Тип доступа:** публичный  

### Основные блоки:
- Список бьюти-процедур  
- Фильтры по категориям (Волосы, Ногти, Брови/Ресницы, Косметология)  
- Поиск по названию услуги  
- Пагинация  

**Динамический контент:** `GET /api/services`

---

## **Страница: Детальная (/services/:id)**
**Тип доступа:** публичный  

### Основные блоки:
- Фото процесса или результата  
- Описание услуги и используемых материалов  
- Цена и примерная длительность  
- Кнопка «Выбрать время / Записаться»  

**Динамический контент:** `GET /api/services/:id`

---

## **Страница: Личный кабинет (/profile)**
**Тип доступа:** приватный  

### Основные блоки:
- Имя и телефон клиента  
- Статус программы лояльности (если есть)  
- Блок «Предстоящие записи» с возможностью отмены  
- Блок «История визитов»  
- Кнопка «Редактировать профиль»  

**Динамический контент:** `GET /check_user_info` и `GET /api/appointments/me`

---

## **Страница: Избранное (/favorites)**
**Тип доступа:** приватный  

### Основные блоки:
- Сохраненные услуги для быстрого доступа  
- Любимые мастера  

---

## **Страница: Онлайн-запись (/booking)**
**Тип доступа:** приватный

- Шаг 1: Выбор услуги (если не выбрана ранее)  
- Шаг 2: Выбор мастера  
- Шаг 3: Интерактивный календарь (выбор даты и доступного времени)

**Тип доступа:** приватный (только admin)

#  **Схема данных и API**

## **Модель User (таблица: `users`)**

| Поле | Тип | Описание |
|------|------|-----------|
| **id** | INT, PK, AUTO_INCREMENT | Уникальный идентификатор клиента |
| **login** | VARCHAR | Логин пользователя (уникальный) |
| **password** | VARCHAR | Хешированный пароль (bcrypt) |
| **phone** | VARCHAR | Номер телефона для связи (уникальный) |
| **role** | VARCHAR | Роль (`client` или `admin`) |

### Особенности модели:
- Пароль хранится **только в виде bcrypt‑хеша** - Логин и телефон проходят строгую валидацию  
- Роль определяет доступ к админ‑панели и управлению расписанием  

---

## **Модель Service (таблица: `services`)**

| Поле | Тип | Описание |
|------|------|-----------|
| **id** | INT, PK, AUTO_INCREMENT | Уникальный идентификатор услуги |
| **title** | VARCHAR | Название процедуры |
| **category** | VARCHAR | Категория (nails, hair, face) |
| **price** | DECIMAL | Стоимость услуги |
| **duration** | INT | Длительность в минутах |
| **image** | VARCHAR | URL изображения или иконки |

### Особенности модели:
- `duration` (длительность) критически важна для расчета свободных окон при онлайн-записи  
- `category` используется для удобной фильтрации на фронтенде
